# Security and sensitive-content reporting

Please do not open a public issue for:

- A benchmark behavior that could materially enable self-harm or violence
- Leaked credentials or personal data
- A way to identify a research participant
- A vulnerability in evaluation infrastructure

Use GitHub's private vulnerability reporting for this repository. If that
feature is unavailable, contact the maintainer through the private contact
method on their GitHub profile.

For ordinary benchmark disagreements, data errors, and documentation issues,
open a public GitHub issue.

This repository does not provide crisis or clinical services.
